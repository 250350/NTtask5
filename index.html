1.1 DNS System

1.1.1 List of Top-Level Domains

  • Generic TLDs (gTLDs): .com, .org, .net, .info, .biz
  • Sponsored TLDs (sTLDs): .edu, .gov, .mil, .aero, .museum
  • Country Code TLDs (ccTLDs): .us, .uk, .de
  • Infrastructure TLDs: .arpa

1.1.2 Hierarchical Structure of DNS

The DNS is structured hierarchically into levels:

  • Root Level: Managed by root servers and points to TLD servers.
  • Top-Level Domains (TLDs): Categories like .com, .org.
  • Second-Level Domains: Owned by individuals or organizations, e.g., example.com.
  • Subdomains: Divisions of second-level domains, e.g., www.example.com.

DNS queries proceed through these levels to resolve domain names to IP addresses. DNS delegation assigns authority to specific name servers.

1.1.3 DNS Records and Reverse DNS

  • A Record: Maps a domain to an IPv4 address.
  • AAAA Record: Maps a domain to an IPv6 address.
  • CNAME Record: Creates an alias for another domain.
  • MX Record: Specifies mail exchange servers.
  • TXT Record: Stores arbitrary text, often for validation.
  • PTR Record: Used for reverse DNS (resolving IP to a domain).

1.1.4 Authoritative vs Non-Authoritative Answers

Authoritative Answer: Comes directly from the server responsible for the domain.

Non-Authoritative Answer: Retrieved from a DNS resolver's cache.

1.1.5 DNS Resolver, DNS Cache, and /etc/hosts

  • DNS Resolver: Resolves domain names to IP addresses.
  • DNS Cache: Stores recent queries to speed up resolution.
  • /etc/hosts: A file for manual domain-to-IP mappings.

1.2 Mail System

1.2.1 Protocols: POP3, IMAP, SMTP

  • POP3: Downloads emails to the client and removes them from the server.
  • IMAP: Synchronizes emails between client and server.
  • SMTP: Sends emails using commands like HELO, MAIL FROM, and RCPT TO.

1.2.2 NNTP Protocol

The Network News Transfer Protocol (NNTP) is used for reading and posting messages in newsgroups.

1.2.3 Email Mailbox, Parameters, Aliases, and Address

  • Mailbox: Stores user emails.
  • Parameters: Includes quotas and access permissions.
  • Aliases: Redirect alternative email addresses to the primary mailbox.
  • Address: Format is username@domain.

1.2.4 Email Structure: Header and Separator

The email structure includes a header with metadata like sender and subject, and a separator (blank line) before the body.

1.2.5 Attachments, Email Coding, MIME

  • Attachments: Files included with emails.
  • Email Coding: Uses Base64 or Quoted-Printable for non-text content.
  • MIME: Defines media types for attachments.

1.2.6 Mailing Lists and Email Gate

  • Mailing Lists: Facilitate group communication.
  • Email Gate: Bridges between email systems and other communication methods.

1.2.7 Email Security: SSL, TLS, PGP

  • SSL/TLS: Encrypts emails during transit.
  • PGP: Provides encryption and digital signatures.

1.2.8 Spam, Blacklists, and Spam Filtering

  • Spam: Unsolicited bulk emails.
  • Blacklists: Lists of spamming IPs.
  • Spam Filtering: Uses algorithms to detect and block spam.
